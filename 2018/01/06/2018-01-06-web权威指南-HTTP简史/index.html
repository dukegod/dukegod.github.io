HTTP简史

HTTP 0.9:

  • 客户端/服务器，请求与响应协议
  • ASCII协议，运行在TCP/IP链接之上
  • 设计用来传输超文本文档（HTTP）
  • 服务器与客户端的连接在每次请求之后都会关闭

HTTP 1.x：

  • 持久链接（默认为keep-alive,除非设置connection：close）
  • 分块编译传输
  • 字节范围请求
  • 增强的缓存机制
  • 传输编码及请求管道
  • cookie
  • 多开TCP链接（不支持多路复用）

HTTP 2.0

  • 改进传输性能,实现高吞吐量，低延迟
  • 多路复用（做到同一个连接并发处理多个请求，而且并发请求的数量比HTTP1.1大了好几个数量级）
  • HPACK算法对header的数据进行压缩（二进制协议）

测量工具

针对优化方案：

客户端缓存资源：

  • cache-control 首部用于指定缓存时间
  • last-modified 和 etag 首部提供验证机制

针对HTTP1.x优化建议：

  • 利用http管道
  • 采用域名分区
  • 打包资源减少请求
  • 嵌入小资源

针对HTTP2.0：

  • 服务器的初始为cwnd应该是10个分组
  • 服务器应该通过ALPN协商支持TLS
  • 服务器应该支持TLS恢复以最小的握手延迟
