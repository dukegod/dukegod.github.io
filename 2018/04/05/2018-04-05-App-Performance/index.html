前端性能调优

前端性能调优方案，其实是一个长久的话题，也是在随着互联网以及科技的变化中不断的演变，最近在看一本《高性能网站建设指南》书籍，边做笔记，边查阅相关的资料，边总结下相关的技术环节。

大体技术点如下：

优化方向 优化的方案
请求的数量 合并脚本和样式表，CSS Sprites，拆分初始化负载，划分主域
请求带宽 开启 GZip，精简 Javascript，移除重复脚本，图像优化
缓存利用 使用 CDN，添加 Expires 头，减少 DNS 查找，配置 ETag，使 AjaX 可缓存
页面结构 将样式表放在顶部，将脚本放在底部，尽早刷新文档的输出
代码校验 避免 CSS 表达式，避免重定向

2，HTTP缓存机制

基本的思路就是：Cache-Control,Expires,ETag配合使用

Cache-Control 中指定的缓存过期策略优先级高于Expires，当它们同时存在的时候，后者会被覆盖掉。
ETag 作为与服务器对比验证，在Cache-Control允许的时间内，请求的资源有没有变化。
