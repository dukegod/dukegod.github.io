git 命令与相关的操作

操作流程

每一个提交的时候，先git pull下，以免覆盖其他人的提交信息

git commit =》 git pull =》 git 解决是否有冲突 =》git commit（有冲突提交）=》git push

git status

git status显示你上次提交更新至后所更改或者写入缓存的改动，

git diff 一行一行地显示这些改动具体是啥。

通常执行完 git status 之后接着跑一下 git diff 是个好习惯。
