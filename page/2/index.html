2018-01-06

HTTP简史

HTTP 0.9:

  • 客户端/服务器，请求与响应协议
  • ASCII协议，运行在TCP/IP链接之上
  • 设计用来传输超文本文档（HTTP）
  • 服务器与客户端的连接在每次请求之后都会关闭

HTTP 1.x：

  • 持久链接（默认为keep-alive,除非设置connection：close）
  • 分块编译传输
  • 字节范围请求
  • 增强的缓存机制
  • 传输编码及请求管道
  • cookie
  • 多开TCP链接（不支持多路复用）

HTTP 2.0

  • 改进传输性能,实现高吞吐量，低延迟
  • 多路复用（做到同一个连接并发处理多个请求，而且并发请求的数量比HTTP1.1大了好几个数量级）
  • HPACK算法对header的数据进行压缩（二进制协议）

渲染树资源加载

浏览器网页解析过程 :

1、浏览器通过请求的 URL 进行域名解析，向服务器发起请求，接收文件（HTML、CSS、JS、Images等等）。
2、HTML 文件加载后，开始构建 DOM Tree
3、CSS 样式文件加载后，开始解析和构建 CSS Rule Tree
4、结合DOM和CSSOM，生成一棵渲染树（包含每个节点的视觉信息）
5、将布局绘制（paint）在屏幕上
6、Javascript 脚本文件加载后， 通过 DOM API 和 CSSOM API 来操作 DOM Tree 和 CSS Rule Tree

浏览器渲染 :

1、浏览器引擎通过 DOM Tree 和 CSS Rule Tree 构建 Rendering Tree
2、Rendering Tree 并不与 DOM Tree 对应，比如像 标签内容或带有 display: none; 的元素节点并不包括在 Rendering Tree 中 。
3、通过 CSS Rule Tree 匹配 DOM Tree 进行定位坐标和大小，是否换行，以及 position、overflow、z-index 等等属性，这个过程称为 Flow 或 Layout 。
4、最终通过调用Native GUI 的 API 绘制网页画面的过程称为 Paint 。

移动端开发优化点

移动端对于 a button input textarea 处理方案

  • ios用户点击一个链接，会出现一个半透明灰色遮罩与部分android系统中元素被点击时产生的边框
    如果想要禁用，可设置-webkit-tap-highlight-color的alpha值为0，也就是属性值的最后一位设置为0就可以去除半透明灰色遮罩。

  • -webkit-user-modify有个副作用，就是输入法不再能够输入多个字符。

  • 移除特定元素或整个文档的触发延迟，而无需禁用缩放：



.css{
-webkit-appearance:none;
}

input,
textarea {
　　border: 0; / 方法1 /
　　-webkit-appearance: none; / 方法2 /
}

input::-webkit-input-placeholder{color:#AAAAAA;}
input:focus::-webkit-input-placeholder{color:#eee}

input::-webkit-input-speech-button {display: none}

```
